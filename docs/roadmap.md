# Roadmap

## MVP

- Node.js API with required endpoints
- status page
- storage abstraction
- mock-first intent parser
- automation executor
- MQTT with embedded broker option
- media fallback via HTTP streaming
- optional File Browser probes and wrapper

## Next

- stronger auth model for LAN exposure
- `llama.cpp` validation on the target phone
- better media browsing and indexing
- optional external broker profiles
- richer automation catalog

## Later

- OpenAI adapter as optional fallback
- OTG mount heuristics and auto-refresh
- better log rotation and retention
- packaging scripts tuned for a real Termux deployment

## Intentionally Deferred

- Jellyfin as a hard requirement
- destructive remote file actions
- shell execution from user input
- POS-side SUNMI integration
